# Шейдеры — план обучения

## Стек

**Three.js + ShaderMaterial** — оптимально для старта. Не чистый WebGL (слишком много бойлерплейта), не нодовый редактор (не учит шейдерам). Именно шейдеры руками, но с Three.js как scaffolding.

## Порядок практики

### Шаг 1 — Первый ShaderMaterial

Плоскость, минимальный vertex и fragment шейдер. Просто закрасить в один цвет. Цель — понять как данные вообще передаются, что такое `projectionMatrix * modelViewMatrix * vec4(position, 1.0)`.

### Шаг 2 — Uniforms и время

Передать `uTime` из JS, в fragment шейдере менять цвет через `sin(uTime)`. Увидеть связь CPU → GPU в реальном времени.

### Шаг 3 — UV и градиент

Покрасить плоскость градиентом через `vUv` — без текстур, просто `gl_FragColor = vec4(vUv.x, vUv.y, 0.0, 1.0)`. Понять как UV выглядит визуально.

### Шаг 4 — Деформация вершин

Sin-волна в vertex шейдере через `uTime`. Шарф в простейшем виде.

### Шаг 5 — Нормали и свет

Добавить пересчёт нормали, посчитать простой diffuse свет в fragment шейдере. Увидеть как форма и свет связаны.

### Шаг 6 — Мышка

Передать `uMouse`, добавить локальное смещение вершин. Интерактивность.

### Шаг 7 — Шум

Подключить Simplex noise (готовая GLSL функция, не пишешь сам), заменить sin на шум. Органичное движение.

## Ресурсы

- **The Book of Shaders** — book-of-shaders.com. Лучшее что есть по GLSL, интерактивные примеры прямо в браузере. Читать параллельно с практикой.
- **Shadertoy** — shadertoy.com. Смотреть чужой код, разбирать как сделано. Только fragment шейдеры, без Three.js — но для понимания математики идеально.
- **Three.js примеры** — threejs.org/examples. Искать примеры с ShaderMaterial.

## Как практиковать

Vite + Three.js локально, один файл `index.js`. Никаких фреймворков поверх. Шейдеры держи в отдельных `.glsl` файлах — удобнее читать с подсветкой синтаксиса. В VS Code есть расширение **glsl-canvas** — видишь шейдер сразу при редактировании.
